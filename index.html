REST API

This REST API provides programmatic access to the information gathered and processed by Meshwork. All responses are available in JSON.

Endpoints

Resource URL

http://ec2-52-8-226-123.us-west-1.compute.amazonaws.com:3000

There are currently only two endpoints: /search and /data.

GET /search

Returns a single-element JSON list that contains information about the search query. A response object contains two keys, Data and Neighbors. The Data object contains three keys, PageRank, URL, and VertexId. The Neighbors object contains two keys, FirstDegree and SecondDegree. These neighbors are returned as a list of VertexIds, sorted by order of descending Page Rank. If you want to query only the top K neighbors, it is as simple as getting the first K elements from the returned list.

Parameters

url The URL that you want to query (e.g., google.com). If this parameter is specified, id does not have to be.
id The ID of the URL you want to query. The ID is the integer MD5 hash of the url. If this parameter is specified, url does not have to be.

Example Request

http://ec2-52-8-226-123.us-west-1.compute.amazonaws.com:3000/search?url=google.com

Example Response

[ { "Data": { "PageRank": "840.8668910315986", "URL": "google.com", "VertexId": "492380404" }, "Neighbors": { "FirstDegree": [ "-1706267909", "1046483734", ... ] "SecondDegree": [ ... ] } } ]

GET /data

Returns a single JSON that contains the Data information about the search query. A response object contains information about the Data of the query, which contains three keys: PageRank, URL, and VertexId. It is just a simplified version of the /search endpoint that does not contain information about the Neighbors.

Parameters

url The URL that you want to query (e.g., google.com). If this parameter is specified, id does not have to be.
id The ID of the URL you want to query. The ID is the integer MD5 hash of the url. If this parameter is specified, url does not have to be.

Example Request

http://ec2-52-8-226-123.us-west-1.compute.amazonaws.com:3000/data?url=google.com

Example Response

{ "PageRank": "840.8668910315986", "URL": "google.com", "VertexId": "492380404" }

About

Meshwork is my big data project as part of Insight Data Science's Engineering fellowship program from June 2015 through July 2015.

What is Meshwork?

If you own a website, then you own a piece of the vast internet web graph. When you look at your website's relationships (links) to other websites, then those relationships can be viewed, as what I like to call, a mesh in the internet network. So, Mesh + Network = Meshwork.

The main purpose of the project was to gain experience in processing large amounts of data - the volume in the three V's (volume, velocity, variety). The end product might be interesting for SEO analysis, to see which are the most credible websites that are affecting your own website's Page Rank.
